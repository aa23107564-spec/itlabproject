# 如何调整第一章的灯光

## ⚠️ 重要说明

**配置文件已经在工作！** 但由于 webpack 模块缓存，修改配置后需要**硬刷新浏览器**才能生效。

## ✅ 已确认工作的配置

从控制台日志确认，以下配置**确实被应用**了：

```
配置版本: 35
窗plight: 系数 0.001, 最终强度: 7.4216  ✓
窗2plight: 系数 0.001, 最终强度: 7.2797  ✓
吊燈light: 系数 0.00003, 最终强度: 0.2387  ✓
（所有8个灯光都正确加载）
```

## 🔧 正确的调整步骤

### 步骤1：修改配置
编辑 `src/components/Chapter1LightConfig.js`

### 步骤2：保存文件
按 `Ctrl + S`

### 步骤3：硬刷新浏览器（必须！）
**按 Ctrl + Shift + R**（不是普通的 Ctrl + R）

或者：
1. 按 F12 打开开发者工具
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 步骤4：等待5秒
让场景完全加载

### 步骤5：查看效果
观察场景变化

## 🎯 测试案例

试试这个来确认配置工作：

### 测试1：大幅提高吊灯亮度
```javascript
// 在 Chapter1LightConfig.js 中
configVersion: 37,  // +1
'吊燈light': 0.01,   // 从 0.00003 改成 0.01（提高333倍）
```

保存 → 硬刷新 → 你应该看到吊灯变得**非常非常亮**

### 测试2：降低窗户灯光
```javascript
configVersion: 38,  // +1
'窗plight': 0.0001,   // 从 0.001 改成 0.0001（降低10倍）
'窗2plight': 0.0001,
```

保存 → 硬刷新 → 窗户的蓝光应该变暗

## 💡 为什么感觉调整没效果？

### 原因1：调整幅度太小
```javascript
// 不明显
'吊燈light': 0.00003 → 0.00005  (只提高67%)

// 明显
'吊燈light': 0.00003 → 0.001    (提高33倍)
```

### 原因2：灯光原本就很弱
```javascript
// 这些灯光本身就很弱，调整后差异不明显：
'吊燈light': 最终 0.24
'檯燈light': 最终 0.52
'吉他light': 最终 1.16

// 这些灯光强，调整明显：
'窗plight': 最终 7.42
'窗2plight': 最终 7.28
```

### 原因3：没有硬刷新
普通刷新（Ctrl+R）不会清除模块缓存，必须硬刷新（Ctrl+Shift+R）

## 🚀 **现在请按以下步骤测试**

### 极端测试：把吊灯调到最大
```javascript
// 在 Chapter1LightConfig.js 中
configVersion: 37,
'吊燈light': 0.1,  // 调到最大！
```

**保存 → 硬刷新（Ctrl+Shift+R）→ 场景应该超级亮，吊灯发出强光！**

如果这样都没有变化，说明：
1. 浏览器缓存太顽固
2. 需要关闭浏览器重新打开
3. 或者服务器没有正确重启
